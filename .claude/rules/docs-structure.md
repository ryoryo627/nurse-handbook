# ページ構造ルール

## テンプレート

新しいページは `docs/99_templates/page.md` をコピーして作る。

## ページの型（順番）

1. **まずこれだけ**: 2〜4行で要点
2. **このページのゴール**: ゴールの姿
3. **やり方**: 手順や方法
4. **例外**: 特別なケース
5. **相談**: 誰に・いつまでに・何を言う
6. **禁止**: やってはいけないこと
7. **チェック**: 確認リスト
8. **関連**: 用語集や関連ページ

不要なセクションは省略可。順番は守る。

## frontmatter（必須）

```yaml
---
title: "ページ名"
who: "読む人（看護主任・師長など）"
owner: "責任者（看護部○○担当など）"
last_checked: "2025-12-23"
---
```

### frontmatter ルール

- 4項目すべて必須
- `last_checked` は編集したら今日の日付に更新
- `who` は対象読者を明記
- `owner` は責任部署を明記

## nav への追加

`mkdocs.yml` の `nav:` セクションに追加する。

```yaml
nav:
  - セクション名:
    - フォルダ/ファイル名.md
```

追加後は `mkdocs build` でエラーなし確認。

## 例外

- 用語集（`glossary.md`）: この型に従わなくてよい
- テンプレート（`99_templates/`）: 型の見本なのでそのまま
- インデックス（`index.md`）: 簡略化可
