# ページ構造ルール

docs/ 配下の Markdown ファイルに適用。

## frontmatter（必須）

すべてのページの先頭に書く:

```yaml
---
title: "ページ名"
who: "読む人（看護主任・師長など）"
owner: "責任者（看護部○○担当など）"
last_checked: "2025-12-23"
---
```

## ページの型

この順番で書く（不要なものは省略可）:

1. **まずこれだけ** - 2〜4行で要点
2. **このページのゴール** - ゴールの姿を3〜5個
3. **やり方** - 手順や方法
4. **例外** - 特別なケース
5. **相談** - 誰に／いつまでに／何を言う
6. **禁止** - やってはいけないこと
7. **チェック** - 確認リスト
8. **関連** - 用語集や関連ページへのリンク

## 新しいページを作るとき

1. `docs/99_templates/page.md` をコピー
2. ファイル名を決める（例: `new-topic.md`）
3. frontmatter を更新
4. 内容を書く
5. mkdocs.yml の nav に追加

## 用語集へのリンク

- 専門語の初出時のみリンクする
- リンク形式: `[用語名](../00_start/glossary.md#anchor-id)`
- アンカーIDは小文字英語とハイフン

## 例外

- 用語集はページの型に従わなくてよい
- テンプレートページは型の見本なのでそのまま
