# 文章の書き方ルール

## 中学2年生語彙ルール

| やること | やらないこと |
|----------|--------------|
| 中学2年生でもわかる言葉 | ビジネス用語を並べる |
| 専門語は用語集にリンク | 説明なしで略語を使う |
| 初出だけリンクする | 同じ言葉に何度もリンク |

### NG → OK 例

- NG: 「ステークホルダーにアラインする」
- OK: 「関係者と方向を合わせる」

## 見出しルール

| やること | やらないこと |
|----------|--------------|
| 短くする | 長くする |
| 「目的」と書く | 「目的（何のため？）」と書く |
| 1つの意味で書く | 括弧で補足を入れる |

### NG → OK 例

- NG: 「相談（誰に聞けばいい？）」
- OK: 「相談」

## 文章ルール

- 1文は短く
- 箇条書きを使う
- 絵文字は見出しに使わない

## 「相談」セクションの書き方

必ず3点を書く：

- **誰に**: 相談先
- **いつまでに**: 期限
- **何を言う**: 伝える内容

## 用語集リンク（絶対遵守）

- 用語集: `docs/00_start/glossary.md`
- リンク形式: `[用語名](../00_start/glossary.md#anchor)`
- **ページ内初出の単語のみリンク**（同じページで2回目以降は絶対にリンクしない）

### リンクの例

```markdown
○ 最初の出現：[Issue](glossary.md#issue)を送信する
× 2回目以降：Issueを送信する（リンクなし）
```

## 用語集の書き方ルール（絶対遵守）

用語集に用語を追加する際は、以下を必ず守る：

### 1. 英単語には読み方を併記

英単語や外来語には、必ずカタカナで読み方（発音）を併記する。

```markdown
○ ## Issue（イシュー） {#issue}
× ## Issue {#issue}

○ ## API（エーピーアイ） {#api}
× ## API {#api}
```

### 2. 例は使用例を書く

「例」には、その用語がどう使われるかの使用例を書く。
定義文や説明文は書かない。

```markdown
○ **例**：ハンドブックに間違いを見つけたとき、各ページの下部からIssueを送信する
× **例**：問題を報告する仕組みのこと
× **例**：気づいたことを伝えるために使う
```

### 3. 必須項目

すべての用語は以下の3つを必ず含める：

- **意味**：その用語の意味
- **例**：実際の使用例（定義文ではない）
- **言いかえ**：別の言い方や類義語

```markdown
## Issue（イシュー） {#issue}

**意味**：気づいたことや提案を報告する仕組み

**例**：ハンドブックに間違いを見つけたとき、各ページの下部からIssueを送信する

**言いかえ**：提案、問題報告
```
