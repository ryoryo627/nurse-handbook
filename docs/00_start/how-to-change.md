# 変更の仕方

## まずこれだけ

やり方を変えるときは、**先にハンドブックを直して**から、現場で実行します。
変更は[PR](glossary.md#pr)で行います。

## できた状態

- 変更の理由と開始日が記録に残る
- 関係する人がレビューしてから反映される
- いつ・誰が・何を変えたかがわかる

## やり方

### 1. 変えたいことを整理する

- 今のやり方（ハンドブックの記述）
- 新しいやり方（こう変えたい）
- 変える理由（なぜ変えるのか）
- いつから変えるか

### 2. PRを出す

1. ハンドブックのファイルを編集する
2. PRを作成する
3. テンプレートに沿って記入する
4. レビュー担当者に通知が行く

### 3. レビューを受ける

- 責任者がレビューする
- 必要なら修正する
- 承認されたらマージする

### 4. 現場に伝える

- 変更内容を関係者に伝える
- 開始日から新しいやり方で動く

## 例外

- 誤字脱字の修正は、レビューなしでマージしてよい
- 緊急の変更は、先に口頭で伝えて、あとからPRを出す

## 相談

- **誰に**：そのページの責任者（[責任者一覧](owners.md)を見る）
- **いつまでに**：変更したい日の1週間前まで
- **何を言う**：「○○を○○に変えたい。理由は○○」

## 禁止

- 現場だけ変えてハンドブックを放置すること
- 複数の場所に同じルールを書くこと（必ず1か所だけ）
- 理由を書かずに変更すること
