# 変更の仕方

## まずこれだけ

気になるページを開いて、ページ下部の**「このページについて」**から修正案を送ってください。

## このページのゴール

- 変更したい内容が記録に残る
- 責任者が内容を確認して対応する
- いつ・誰が・何を報告したかがわかる

## やり方

### 1. 気になるページを開く

- ハンドブックで変更したい、または間違いを見つけたページを開く

### 2. ページ下部の「このページについて」を見つける

- すべてのページの一番下に、[Issue](glossary.md#issue)送信ボックスがあります
- 2種類のボタンがあります：
  - **間違いを報告**：誤字・脱字、内容の間違いを見つけたとき
  - **追加・改善を提案**：新しい内容の追加や、やり方の改善を提案したいとき

### 3. 該当するボタンをクリック

- GitHubのフォームが開きます
- ページ名は自動で入力されます

### 4. フォームに記入して送信

#### 間違いを報告する場合
- **どこが間違っていますか**：該当箇所を具体的に書く
- **正しくはどうあるべきですか**：正しい内容を書く
- **補足情報**：必要に応じて追加情報を書く

#### 追加・改善を提案する場合
- **どんな内容を追加・改善したいですか**：提案内容を具体的に書く
- **なぜ必要ですか**：理由や背景を書く
- **いつから必要ですか**：開始希望日を書く（わかれば）

### 5. 責任者が対応する

- 送信した報告は、そのページの責任者に通知されます
- 責任者が内容を確認して、必要な対応を行います
- 対応が完了したら、報告がクローズされます

## 例外

- **緊急の変更**：先に口頭で責任者に伝えて、あとから報告を送る
- **大きな変更**：複数ページにまたがる場合は、責任者に直接相談する

## 相談

- **誰に**：そのページの責任者（[責任者一覧](owners.md)を見る）
- **いつまでに**：変更したい日の1週間前まで
- **何を言う**：「○○ページの○○を○○に変えたい。理由は○○」

## 禁止

- 現場だけ変えてハンドブックを放置すること
- 複数の場所に同じルールを書くこと（必ず1か所だけ）
- 理由を書かずに変更を提案すること

## チェック

- [ ] どこを変えたいか具体的に書いたか
- [ ] 変更理由を書いたか
- [ ] 必要に応じて開始希望日を書いたか
