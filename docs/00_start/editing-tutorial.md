# 編集者向けチュートリアル

## まずこれだけ

このページでは、報告された[Issue](glossary.md#issue)を確認し、ハンドブックの記事を修正・追加する方法を説明します。
パソコンの基本操作ができれば、特別な知識がなくても編集できます。

---

## このページのゴール

- 報告されたIssueを確認できる
- 記事の間違いを修正できる
- 新しい記事を追加できる
- 基本的な書き方がわかる

---

## 1. GitHubとは

[GitHub](glossary.md#github)は「みんなで文書を管理する場所」です。

- ハンドブックの記事は、すべてGitHubに保存されています
- 誰かが修正すると、いつ・誰が・何を変えたか記録が残ります
- 間違えても元に戻せるので安心です

### アカウントを作る

編集するにはGitHubアカウントが必要です。

1. [github.com](https://github.com) を開く
2. 「Sign up」をクリック
3. メールアドレスとパスワードを入力
4. 画面の指示に従って登録を完了

---

## 2. Issueを確認する

Issueは「報告された内容」のことです。読者から送られた間違いの報告や改善提案がここに集まります。

### 確認の手順

1. [ハンドブックのGitHubページ](https://github.com/ryoryo627/nurse-handbook) を開く

2. 上のメニューから「**Issues**」をクリック
   - 数字が表示されていたら、それが未対応の報告の数です

3. 一覧から確認したい報告をクリック
   - タイトルの先頭に [報告] と付いています

4. 内容を確認する
   - **報告の種類**：間違いの報告か、追加・改善の提案か
   - **ページ**：どのページについてか
   - **内容**：具体的に何を修正してほしいか

### 対応が終わったら

1. Issueのページで「**Close issue**」をクリック
2. これで対応完了として記録されます

---

## 3. 記事を修正する

GitHubの画面から直接修正できます。

### 修正の手順

1. [ハンドブックのGitHubページ](https://github.com/ryoryo627/nurse-handbook) を開く

2. 修正したいファイルを探す
   - 記事は `docs` フォルダの中にあります
   - フォルダ名をクリックして中を見る
   - 例：「人が足りない」は `docs/10_troubles/not-enough-people.md`

3. ファイル名をクリックして開く

4. 右上の **鉛筆マーク**（Edit this file）をクリック

5. 内容を修正する
   - 直接文字を書き換えます
   - 書き方は「5. 基本的な書き方」を参考に

6. 下にスクロールして「**Commit changes**」をクリック

7. 変更内容を書く
   - 例：「電話番号を修正」「手順3を追加」
   - 「Commit changes」ボタンをクリックして[Commit](glossary.md#commit)する

8. これで修正完了です
   - 数分後にハンドブックに反映されます

---

## 4. 新しい記事を追加する

### 追加の手順

1. [ハンドブックのGitHubページ](https://github.com/ryoryo627/nurse-handbook) を開く

2. 記事を追加するフォルダを開く
   - 例：困ったときの記事なら `docs/10_troubles`

3. 「**Add file**」→「**Create new file**」をクリック

4. ファイル名を入力
   - 英語で、単語はハイフンでつなぐ
   - 最後に `.md` をつける
   - 例：`night-shift-trouble.md`

5. 内容を書く
   - [ページテンプレート](../99_templates/page.md)を開く
   - 内容をすべてコピーして貼り付ける
   - 必要な部分を書き換える

6. 「**Commit new file**」をクリックしてCommitする

7. メニューファイルを更新する（次の「メニューに追加する」を見る）

### メニューに追加する

新しい記事を作ったら、メニューにも追加します。

1. `mkdocs.yml` ファイルを開く（一番上の階層にあります）

2. 鉛筆マークをクリックして編集

3. `nav:` の下にある該当セクションを探す
   - 例：困ったときなら `- 困ったとき:` の下

4. 新しい行を追加
   ```
       - 10_troubles/night-shift-trouble.md
   ```
   - 先頭のスペース（半角4つ）を合わせる

5. 「Commit changes」をクリックしてCommitする

## 5. 基本的な書き方

記事は「[Markdown](glossary.md#markdown)」という書き方で書きます。
特別なソフトは不要で、普通の文字を書くだけです。

### 見出し

`#` の数で見出しのレベルが変わります。

```
# 大見出し
## 中見出し
### 小見出し
```

### 箇条書き

行の先頭に `-` をつけます。

```
- 項目1
- 項目2
- 項目3
```

### 番号付きリスト

行の先頭に `1.` をつけます。

```
1. 最初にやること
2. 次にやること
3. 最後にやること
```

### 太字

`**` で囲みます。

```
**強調したい言葉**
```

### リンク

`[表示する文字](リンク先)` と書きます。

```
[用語集](glossary.md#issue)
```

### チェックリスト

```
- [ ] まだやっていない
- [x] 完了した
```

### 表

```
| 項目 | 内容 |
|------|------|
| 名前 | 田中 |
| 役職 | 主任 |
```

## 6. よくある質問

### 間違えて修正してしまった

GitHubは変更履歴が残るので、元に戻せます。
責任者に連絡してください。

### 編集ボタンが見つからない

GitHubにログインしていないと表示されません。
右上の「Sign in」からログインしてください。

### 反映されない

変更後、反映まで数分かかります。
5分以上経っても反映されない場合は、責任者に連絡してください。

### 書き方がわからない

- 似たページを参考にする
- [書き方ルール](writing-rules.md)を見る
- 責任者に聞く
